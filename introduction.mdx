MHR-CFW (MasterHttpRelay + Cloudflare Worker) is an open-source tool that lets you access blocked websites by routing your traffic through Google's infrastructure. From the perspective of any DPI (Deep Packet Inspection) filter on the network, your connection looks like normal HTTPS traffic to `www.google.com` — while the actual destination stays hidden inside the relay.

## How it works at a glance

Your browser connects to a local proxy running on your machine. That proxy forwards requests through Google Apps Script — fronted by `www.google.com` — to a Cloudflare Worker, which fetches the real destination and returns the response through the same path.

```
Browser → Local Proxy → www.google.com (SNI) → Google Apps Script → Cloudflare Worker → Target
```

The network's DPI filter sees only a TLS handshake to `www.google.com`. The encrypted payload carries the real request to `script.google.com`, which your deployed relay handles.

## Get started in 4 steps

<Steps>
  <Step title="Deploy the Cloudflare Worker">
    Create a free Cloudflare Workers account and deploy `script/worker.js` as your exit node. This worker receives requests from the Apps Script relay and fetches the actual destination URLs.
  </Step>
  <Step title="Deploy the Google Apps Script relay">
    Create a new Google Apps Script project, paste in `script/Code.gs`, set your `AUTH_KEY` and `WORKER_URL`, then deploy it as a web app accessible to anyone.
  </Step>
  <Step title="Configure and start the local proxy">
    Clone the repository, install Python dependencies, and run `python setup.py` to create your `config.json`. Then start the proxy with `python main.py` or `run.bat` / `run.sh`.
  </Step>
  <Step title="Point your browser at the proxy">
    Configure your browser or system to use the local HTTP proxy at `127.0.0.1:8085` (or the SOCKS5 proxy at `127.0.0.1:1080`). Verify your connection at [ipleak.net](https://ipleak.net) — you should see a Cloudflare IP address.
  </Step>
</Steps>
